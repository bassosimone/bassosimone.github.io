TCP Pipeline Filling

This demo visualizes how TCP fills a network pipe, using packet-level traces captured from a gVisor userspace network simulation.

Network tab

Shows the network topology with animated packet replay. Each packet event highlights the relevant network link and plays an audio tone. Step through packets one at a time, or press fast forward to replay the entire trace with real inter-packet timing (scaled 50x).

Audio sonification

Each network node is mapped to a pentatonic note (C4, D4, E4, G4). Waveforms vary by packet type:

Sine (smooth)
Data packets
Triangle (soft)
Control packets (SYN, FIN, ACK-only)
Sawtooth (harsh)
RST packets

"Entered" events play at the base frequency; "delivered" events play one octave higher. The result is an audible picture of the connection: the slow handshake tones followed by a rapid cascade as the pipe fills.

Packets tab

A sortable packet table with Wireshark-style protocol dissection. Use the perspective buttons to filter by network node. Click any packet to see its decoded headers (IP, TCP, UDP, HTTP, TLS, DNS).

Source

Extracted from the Censorship Lab, a Go backend + vanilla JS frontend that runs gVisor TCP/IP simulations with a configurable DPI engine.
